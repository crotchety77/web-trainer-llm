# Образовательная веб-платформа с интегрированным ИИ-ассистентом (Web Trainer Platform)

Интерактивная образовательная веб-платформа для обучения программированию, объединяющая структурированный учебный процесс и интеллектуальную поддержку на базе большой языковой модели (**YandexGPT**). 

Система предоставляет студентам полноценную среду для изучения теории и практики написания кода, а преподавателям (авторам курсов) — автоматизированные инструменты проектирования учебных материалов.

---

## 🚀 Ключевые возможности и сценарии использования

### 🧑‍🎓 Для Студента
*   **Интерактивный учебный кабинет:** Изучение лекционных материалов, отслеживание прогресса прохождения курса с визуальным индикатором (ProgressBar).
*   **Интерактивный редактор кода:** Написание решений задач непосредственно в браузере с помощью встроенного **Monaco Editor** (движок VS Code) с подсветкой синтаксиса и автоформатированием.
*   **Изолированная автопроверка (Docker Sandbox):** Мгновенная компиляция и прогон решений на тест-кейсах в безопасной песочнице (**Piston API**).
*   **Защита от подгонки кода:** Поддержка *скрытых тест-кейсов*, детали которых скрыты от студента, что стимулирует написание алгоритмически верных решений.
*   **Интеллектуальный ИИ-чат-ассистент:** Сопровождение студента на протяжении всего урока с возможностью переключения специализированных режимов:
    *   `Базовый режим (default)` — Наставничество по теории урока в формате сократовского диалога.
    *   `Помощь с кодом (code_help)` — Диагностика и локализация ошибок компиляции и тестов без прямой выдачи готового кода.
    *   `Объяснить теорию (explain)` — Декомпозиция сложных алгоритмов и концепций простыми словами.
    *   `Пример (example)` — Подбор бытовых аналогий и генерация чистых примеров кода.
    *   `Быстрая справка (search_info)` — Лаконичное извлечение справочных данных (синтаксис функций, библиотеки).

### ✍️ Для Автора (Преподавателя)
*   **Конструктор курсов:** Визуальный интерфейс для добавления уроков, сортировки их структуры (Drag-and-Drop) и редактирования метаданных.
*   **Конструктор заданий:** Проектирование практических задач по программированию, указание стартового шаблона кода, конфигурация тест-кейсов (входных и ожидаемых выходных данных).
*   **Встроенное тестирование эталона:** Написание и запуск эталонного решения автором в конструкторе для моментальной валидации добавленных тест-кейсов.
*   **ИИ-методист:** Интерактивный генератор учебного контента на базе ИИ:
    *   `Улучшить стиль (improve_text)` — Стилистическая и грамматическая редактура сырых лекций.
    *   `Создать задание (generate_task)` — Автогенерация задач по программированию или тестов (квизов с правильными ответами, дистракторами и подсказками) на основе текста лекции.
    *   `Составить план (structure)` — Логическое секционирование длинного материала и определение точек контроля.

### 🛡️ Безопасность и администрирование
*   **JWT-аутентификация** с безопасным хранением токенов на клиенте (`localStorage`) и валидацией на бэкенде.
*   **Ролевая модель доступа (RBAC):** Разграничение эндпоинтов и разделов интерфейса для ролей `Студент`, `Автор` и `Администратор`.
*   **Панель Администратора:** Управление учетными записями пользователей, изменение их ролей и параметров доступа.

## 🛠️ Технологический стек

### Фронтенд (Client)
*   **React 18** (Single Page Application) + **Vite** (инструмент сборки).
*   **React Router v6** (клиентская маршрутизация и защита приватных путей `ProtectedRoute`).
*   **Monaco Editor** (`@monaco-editor/react`) для написания кода.
*   **Vanilla CSS** для кастомного, адаптивного интерфейса (поддерживает десктопные и мобильные оверлей-меню).
*   **React Markdown** для рендеринга теории и ИИ-ответов с форматированием.

### Бэкенд (Server)
*   **Node.js** + **Express.js** (REST API сервер с асинхронным вводом-выводом).
*   **PostgreSQL** (хранение данных о пользователях, курсах, уроках, прогрессе и квизах).
*   **Piston API** (локально развернутый Docker-контейнер для компиляции кода).
*   **YandexGPT API** (библиотека интеграции ИИ с шаблонизацией промптов `promptBuilder`).

## ⚙️ Установка и запуск проекта

### Вариант А. Быстрый запуск через Docker Compose (Рекомендуемый)
Убедитесь, что у вас установлен Docker и запущен Docker Daemon.

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Создайте файл `.env` в корневой папке на основе примера:
   ```bash
   cp .env.example .env
   ```
   *Заполните переменные доступа к YandexGPT API, если планируется использование ИИ-ассистента.*
3. Запустите сборку и старт всех сервисов одной командой:
   ```bash
   docker-compose up --build
   ```
4. Приложение будет доступно по адресам:
   *   Клиентская часть: `http://localhost:3000`
   *   Серверная часть: `http://localhost:5000`
   *   Песочница Piston: `http://localhost:2001`

---

### Вариант Б. Локальный запуск в режиме разработки (Manual)

#### 1. Подготовка Базы Данных (PostgreSQL)
*   Создайте базу данных с именем `web_trainer_platform`.
*   Импортируйте структуру таблиц из файла `schema.sql`:
    ```bash
    psql -U postgres -d web_trainer_platform -f schema.sql
    ```

#### 2. Настройка Бэкенда (`server`)
1. Перейдите в каталог сервера:
   ```bash
   cd server
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Создайте локальный файл `.env` в папке `server/` и укажите параметры подключения:
   ```env
   PORT=5000
   DATABASE_URL=postgresql://postgres:пароль@localhost:5432/web_trainer_platform
   JWT_SECRET=ваш_секретный_ключ
   PISTON_URL=http://localhost:2000 # Ссылка на локальный/удаленный Piston
   # Параметры авторизации YandexGPT (ключи)
   ```
4. Запустите сервер в режиме разработки:
   ```bash
   npm run dev
   ```

#### 3. Настройка Фронтенда (`client`)
1. Перейдите в каталог клиента:
   ```bash
   cd ../client
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите dev-сервер:
   ```bash
   npm run dev
   ```
   *Фронтенд запустится на порту `5173` (`http://localhost:5173`).*

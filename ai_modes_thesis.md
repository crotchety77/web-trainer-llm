# Приложение. Режимы функционирования и шаблоны промптов ИИ-ассистента

В разработанной образовательной платформе логика взаимодействия с ИИ-ассистентом базируется на ролевой модели доступа и специализированных когнитивных режимах (сценариях). Это позволяет адаптировать поведение большой языковой модели (LLM) под текущие задачи конкретного пользователя — будь то студент, проходящий практику, или автор, проектирующий курс.

Настройка поведения ИИ-ассистента реализуется на стороне сервера путём динамической подстановки системных инструкций (системных промптов) в зависимости от выбранного в пользовательском интерфейсе режима и роли авторизованного субъекта.

Ниже приведены классификация и подробное описание режимов работы ИИ-ассистента, оформленные в виде таблиц для интеграции в пояснительную записку дипломного проекта.

---

### Таблица 3.X — Режимы работы ИИ-ассистента для роли «Студент»

| Название режима (Label в UI) | Идентификатор (`id`) | Описание логики работы и системных ограничений (для пояснительной записки) |
| :--- | :--- | :--- |
| **Базовый режим** | `default` | **Сценарий общего интерактивного наставничества.** Активируется по умолчанию, когда не выбран ни один специализированный фильтр. Модель ориентирована на ведение контекстного диалога по теме текущего урока без прямой выдачи программного кода. Она отвечает на общие вопросы по лекционному материалу и наталкивает студента на самостоятельный поиск решений. |
| **Помощь с кодом** | `code_help` | **Интеллектуальная диагностика и отладка программного кода.** Модель анализирует переданный код студента, лог вывода Docker-песочницы Piston API (ошибки компилятора, runtime-исключения, результаты тестирования). Ассистент локализует место возникновения сбоя, объясняет причину ошибки и формулирует наводящие вопросы. Действует строгое системное ограничение: модели запрещено предоставлять готовые фрагменты исправленного кода, что стимулирует самостоятельную когнитивную деятельность обучающегося. |
| **Объяснить теорию** | `explain` | **Адаптивное разъяснение теоретических концепций и алгоритмов.** Модель берет на себя роль терпеливого преподавателя: декомпозирует сложные термины, структуры данных и шаблоны проектирования на более простые составляющие, пошагово описывает логическую суть процессов и адаптирует уровень сложности изложения под базовые знания студента, избегая избыточной академической терминологии. |
| **Пример** | `example` | **Ассоциативное обучение через генерацию примеров и аналогий.** Ассистент подбирает наглядные аналогии из реального мира (физические процессы, бытовые ситуации) для абстрактных концепций программирования (классы, интерфейсы, асинхронность), а также генерирует простые изолированные примеры кода, демонстрирующие синтаксис изучаемого языка в чистом виде. |
| **Быстрая справка** | `search_info` | **Предоставление точной технической справочной информации.** Режим оптимизирован для быстрого извлечения конкретных фактов без ведения развернутого диалога. Модель лаконично формулирует ответы, содержащие синтаксические конструкции встроенных функций, структуры библиотек, типы данных или терминологические стандарты, используя Markdown-таблицы и блоки кода для ускорения восприятия. |

### Таблица 3.Y — Режимы работы ИИ-ассистента для роли «Автор»

| Название режима (Label в UI) | Идентификатор (`id`) | Описание логики работы и системных ограничений (для пояснительной записки) |
| :--- | :--- | :--- |
| **Базовый режим** | `default` | **Методическое проектирование учебного контента.** Модель функционирует в роли универсального ИИ-помощника для автора курса: помогает планировать структуру будущих уроков, набрасывать идеи практических задач, формулировать учебные цели и создавать черновики разделов. |
| **Улучшить стиль** | `improve_text` | **Стилистическая, орфографическая и структурная редактура материалов.** ИИ-ассистент выступает в роли профессионального корректора: перерабатывает сырой текст лекции автора, упрощает перегруженные синтаксические конструкции для улучшения восприятия студентами, исправляет грамматические и пунктуационные опечатки, адаптируя стиль повествования под научно-популярный образовательный формат. |
| **Создать задание** | `generate_task` | **Автоматизированное проектирование контрольно-измерительных материалов.** На основе переданного текста лекции или выбранного контекста модель разрабатывает практические задачи по программированию (включая описание задачи, эталонный алгоритм и примеры тест-кейсов) либо генерирует тестовые вопросы с одиночным/множественным выбором ответа, правильными вариантами, дистракторами (неверными вариантами) и наводящими подсказками для студентов. |
| **Составить план** | `structure` | **Логическое секционирование и структурирование материала.** Анализируя хаотичный или избыточный теоретический материал автора, ассистент предлагает оптимальный сценарий урока: разбивает материал на последовательные логические блоки, формирует заголовки подразделов, выделяет ключевые тезисы в виде списков и определяет наиболее подходящие точки для внедрения практических задач или тестов. |

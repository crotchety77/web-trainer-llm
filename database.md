# Описание структуры базы данных учебной платформы

Данный раздел посвящен описанию спроектированной реляционной базы данных для веб-платформы обучения программированию. В качестве СУБД выбрана объектно-реляционная система **PostgreSQL**, обеспечивающая высокую надежность, производительность и поддержку современных стандартов SQL, а также эффективную работу с полуструктурированными данными формата JSONB.

---

## 1. Сводная таблица сущностей базы данных

В таблице ниже представлена общая информация о структуре таблиц базы данных, их назначении и взаимосвязях.

| № | Имя таблицы в БД | Логическое имя (сущность) | Назначение и использование | Связи с другими таблицами |
| :--- | :--- | :--- | :--- | :--- |
| **1** | `users` | Пользователи | Хранение учетных записей (студенты, авторы курсов, администраторы) и их ключей доступа к ИИ. | Родительская для `courses`, `submissions`, `user_course_progress`, `user_quiz_attempts`, `enrollments`. |
| **2** | `courses` | Учебные курсы | Хранение информации об учебных курсах платформы. | Ссылается на `users` (`author_id`). Родительская для `lessons`, `enrollments`, `user_course_progress`. |
| **3** | `lessons` | Уроки | Логическая группировка учебного материала (модулей/уроков) внутри курса. | Ссылается на `courses` (`course_id`). Родительская для `lesson_blocks`, `user_course_progress`. |
| **4** | `lesson_blocks` | Шаги (блоки) урока | Хранение атомарного контента уроков (теория, практика, тесты). | Ссылается на `lessons` (`lesson_id`). Родительская для `submissions`, `user_course_progress`, `user_quiz_attempts`. |
| **5** | `submissions` | Отправки решений | История и результаты выполнения студентами практических заданий с кодом. | Ссылается на `users` (`student_id`), `lesson_blocks` (`block_id`). |
| **6** | `user_course_progress` | Прогресс обучения | Фиксация завершенных блоков курсов пользователями для расчета общей статистики. | Ссылается на `users`, `courses`, `lessons`, `lesson_blocks`. |
| **7** | `user_quiz_attempts` | Попытки прохождения тестов | Детализированные результаты ответов на тесты для сбора академической аналитики. | Ссылается на `users` (`user_id`), `lesson_blocks` (`block_id`). |
| **8** | `enrollments` | Подписки на курсы | Зачисление студентов на определенные курсы (связь «многие-ко-многим»). | Ссылается на `users` (`student_id`), `courses` (`course_id`). |

## 2. Детальная спецификация таблиц (схемы данных)

### 2.1. Таблица `users` (Пользователи)
Предназначена для хранения учетных записей всех участников системы, включая аутентификационные данные, роли и настройки интеграции с ИИ-ассистентом.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор пользователя |
| `name` | `VARCHAR(120)` | `NOT NULL` | Отображаемое имя/фамилия пользователя |
| `email` | `VARCHAR(255)` | `NOT NULL`, `UNIQUE` | Уникальный адрес электронной почты (логин) |
| `password_hash` | `VARCHAR(255)` | `NOT NULL` | Хеш пароля для безопасной авторизации |
| `role` | `VARCHAR(20)` | `NOT NULL`, `CHECK` | Роль: `'student'` (студент), `'author'` (преподаватель), `'admin'` (администратор) |
| `llm_api_key_encrypted` | `TEXT` | `NULL` | Зашифрованный API-ключ для интеграции с YandexGPT |
| `llm_folder_id` | `TEXT` | `NULL` | Идентификатор каталога (Folder ID) в Yandex Cloud |
| `created_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время регистрации пользователя |

### 2.2. Таблица `courses` (Учебные курсы)
Хранит общую информацию об учебных курсах, создаваемых авторами.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор курса |
| `author_id` | `INTEGER` | `NOT NULL`, `REFERENCES users(id) ON DELETE CASCADE` | Идентификатор автора курса (связь с `users`) |
| `title` | `VARCHAR(255)` | `NOT NULL` | Название учебного курса |
| `short_description` | `TEXT` | `NOT NULL`, `DEFAULT ''` | Краткое описание курса для каталога |
| `intro_content` | `TEXT` | `NOT NULL`, `DEFAULT ''` | Развернутое описание/программа курса на главной странице |
| `cover_image_url` | `TEXT` | `NOT NULL`, `DEFAULT ''` | Ссылка на обложку курса |
| `tags_json` | `JSONB` | `NOT NULL`, `DEFAULT '[]'::jsonb` | Полуструктурированный массив тегов курса (для фильтрации) |
| `is_published` | `BOOLEAN` | `NOT NULL`, `DEFAULT FALSE` | Статус публикации (скрыт/виден студентам) |
| `created_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время создания курса |

### 2.3. Таблица `lessons` (Уроки)
Служит для разбиения курса на отдельные логические темы или разделы обучения.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор урока |
| `course_id` | `INTEGER` | `NOT NULL`, `REFERENCES courses(id) ON DELETE CASCADE` | Идентификатор связанного курса (связь с `courses`) |
| `title` | `VARCHAR(255)` | `NOT NULL` | Название урока |
| `position` | `INTEGER` | `NOT NULL`, `DEFAULT 1` | Порядковый номер урока внутри курса для правильной сортировки |
| `created_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время создания урока |

### 2.4. Таблица `lesson_blocks` (Шаги урока)
Ключевая таблица контента, содержащая отдельные шаги урока. Каждый шаг может быть лекцией, тестом или заданием на программирование.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор блока |
| `lesson_id` | `INTEGER` | `NOT NULL`, `REFERENCES lessons(id) ON DELETE CASCADE` | Идентификатор урока (связь с `lessons`) |
| `type` | `VARCHAR(20)` | `NOT NULL`, `CHECK` | Тип шага: `'lecture'` (теория), `'practice'` (практика с кодом), `'test'` (тест) |
| `title` | `VARCHAR(255)` | `NOT NULL` | Заголовок шага |
| `content` | `TEXT` | `NOT NULL`, `DEFAULT ''` | Текстовый контент (лекционный материал, формулировка задания) |
| `quiz_data` | `JSONB` | `DEFAULT '{}'::jsonb` | Конфигурация интерактивного элемента (тесты, автотесты, шаблон кода) |
| `attachment_url` | `TEXT` | `NOT NULL`, `DEFAULT ''` | Метаданные загруженных файлов-методичек (PDF, DOCX) в формате JSON |
| `position` | `INTEGER` | `NOT NULL`, `DEFAULT 1` | Порядковый номер блока внутри урока |
| `created_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время добавления блока |

### 2.5. Таблица `submissions` (Отправки решений)
Регистрирует попытки выполнения практических заданий по программированию, отправленные студентами.

| Имя поля | Tip данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор попытки решения |
| `student_id` | `INTEGER` | `NOT NULL`, `REFERENCES users(id) ON DELETE CASCADE` | Ссылка на студента (связь с `users`) |
| `block_id` | `INTEGER` | `NOT NULL`, `REFERENCES lesson_blocks(id) ON DELETE CASCADE` | Ссылка на блок практического задания (связь с `lesson_blocks`) |
| `code` | `TEXT` | `NOT NULL` | Текст программы, отправленный студентом на проверку |
| `language` | `VARCHAR(40)` | `NOT NULL`, `DEFAULT 'javascript'` | Язык программирования (например, `'javascript'`, `'python'`) |
| `status` | `VARCHAR(40)` | `NOT NULL`, `DEFAULT 'submitted'` | Статус проверки: `'submitted'`, `'executing'`, `'passed'`, `'failed'`, `'error'` |
| `result_message` | `TEXT` | `NOT NULL`, `DEFAULT ''` | Краткое текстовое сообщение компилятора или тестового окружения |
| `tests_result` | `JSONB` | `NOT NULL`, `DEFAULT '{}'::jsonb` | Развернутые результаты прохождения тест-кейсов в формате JSON |
| `created_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Время отправки решения |

### 2.6. Таблица `user_course_progress` (Прогресс обучения)
Служит для фиксации факта прохождения отдельных блоков студентами, что позволяет отслеживать прогресс обучения и формировать статистику.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор записи прогресса |
| `user_id` | `INTEGER` | `NOT NULL`, `REFERENCES users(id) ON DELETE CASCADE` | Ссылка на пользователя (связь с `users`) |
| `course_id` | `INTEGER` | `NOT NULL`, `REFERENCES courses(id) ON DELETE CASCADE` | Дублирующее поле для денормализации (связь с `courses`) |
| `lesson_id` | `INTEGER` | `NOT NULL`, `REFERENCES lessons(id) ON DELETE CASCADE` | Дублирующее поле для денормализации (связь с `lessons`) |
| `block_id` | `INTEGER` | `NOT NULL`, `REFERENCES lesson_blocks(id) ON DELETE CASCADE` | Ссылка на пройденный блок (связь с `lesson_blocks`) |
| `completed_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время успешного завершения шага |

> [!IMPORTANT]
> Для таблицы установлено ограничение уникальности `UNIQUE (user_id, block_id)`, которое предотвращает дублирование прогресса при многократном открытии/прохождении одного и того же шага.

### 2.7. Таблица `user_quiz_attempts` (Попытки прохождения тестов)
Хранит историю прохождения интерактивных тестов студентами.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор попытки тестирования |
| `user_id` | `INTEGER` | `NOT NULL`, `REFERENCES users(id) ON DELETE CASCADE` | Ссылка на студента (связь с `users`) |
| `block_id` | `INTEGER` | `NOT NULL`, `REFERENCES lesson_blocks(id) ON DELETE CASCADE` | Ссылка на тестовый блок (связь с `lesson_blocks`) |
| `answers` | `JSONB` | `NOT NULL`, `DEFAULT '[]'::jsonb` | Список выбранных индексов ответов |
| `is_correct` | `BOOLEAN` | `NOT NULL`, `DEFAULT FALSE` | Флаг успешности прохождения (верны ли все ответы) |
| `created_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Время прохождения попытки |

### 2.8. Таблица `enrollments` (Подписки на курсы)
Реализует связь типа «многие-ко-многим» между студентами и учебными курсами, фиксируя факт зачисления на обучение.

| Имя поля | Тип данных | Ограничения и связи | Описание |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | `PRIMARY KEY` | Уникальный идентификатор зачисления |
| `student_id` | `INTEGER` | `NOT NULL`, `REFERENCES users(id) ON DELETE CASCADE` | Ссылка на студента (связь с `users`) |
| `course_id` | `INTEGER` | `NOT NULL`, `REFERENCES courses(id) ON DELETE CASCADE` | Ссылка на курс (связь с `courses`) |
| `enrolled_at` | `TIMESTAMP` | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время зачисления на курс |

> [!IMPORTANT]
> Уникальное ограничение `UNIQUE (student_id, course_id)` гарантирует невозможность повторного зачисления студента на один и тот же курс.

## 3. Особенности хранения полуструктурированных данных (JSONB)

Использование встроенного в СУБД PostgreSQL типа данных `JSONB` позволило реализовать гибкую и расширяемую схему хранения данных без необходимости создания множества мелких промежуточных таблиц (например, таблиц для хранения вариантов ответов, параметров тест-кейсов или тегов).

Основные сценарии использования `JSONB` в системе представлены в таблице ниже.

| Таблица в БД | Поле | Назначение поля | Пример структуры документа |
| :--- | :--- | :--- | :--- |
| **`courses`** | `tags_json` | Хранение набора текстовых тегов курса для быстрого поиска и фильтрации. | `["JavaScript", "DOM", "Async"]` |
| **`lesson_blocks`** | `quiz_data` | **Для тестов:** варианты ответов, правильность ответов, подсказки.<br>**Для практики:** шаблон начального кода, имя функции, автотесты (входные и ожидаемые значения). | **Тест:**<br>`{"quiz_type": "multiple", "options": [{"text": "Да", "is_correct": true}, {"text": "Нет", "is_correct": false}]}`<br><br>**Практика:**<br>`{"placeholder_code": "function sum(a,b) {}", "test_cases": [{"input": "1 2", "expected_output": "3"}]}` |
| **`submissions`** | `tests_result` | Подробные результаты автоматизированной проверки отправленного студентом кода на сервере. | `{"total": 2, "passed": 1, "failed": 1, "details": [{"input": "1 2", "passed": true}, {"input": "2 3", "passed": false}]}` |
| **`user_quiz_attempts`** | `answers` | Сохранение ответов пользователя, отправленных на проверку (список выбранных индексов опций). | `[0, 2]` |

## 4. Контролируемая денормализация данных

Для оптимизации производительности платформы при высокой нагрузке на этапе проектирования базы данных была применена **контролируемая денормализация** в таблице `user_course_progress`.

### Суть оптимизации:
В таблице `user_course_progress` помимо ссылки на конкретный пройденный блок (`block_id`) избыточно хранятся ссылки на урок (`lesson_id`) и курс (`course_id`).

### Для чего это сделано:
При расчете прогресса студента по курсу или уроку (например, вывод прогресс-бара в личном кабинете) классическая нормализованная схема требовала бы выполнения операции `JOIN` с таблицами `lesson_blocks` и `lessons` для группировки по урокам и курсам. 

Благодаря денормализации, подсчет прогресса сводится к простому запросу:
```sql
SELECT COUNT(*) 
FROM user_course_progress 
WHERE user_id = $1 AND course_id = $2;
```
Этот запрос выполняется мгновенно с использованием созданных индексов, полностью исключая ресурсоемкие операции объединения таблиц (`JOIN`), что значительно ускоряет работу веб-приложения под высокой нагрузкой.

## 5. Обеспечение целостности данных

В базе данных на уровне схемы внедрены механизмы автоматического поддержания целостности:
1. **Каскадное удаление (`ON DELETE CASCADE`):** Использование этого правила для всех внешних ключей гарантирует автоматическую очистку зависимых данных при удалении родительских сущностей (например, при удалении курса автоматически будут удалены все его уроки, шаги, прогресс и подписки студентов, не допуская появления «битых» ссылок).
2. **Ограничения проверки (`CHECK`):** Строго ограничивают допустимые роли пользователей (`role IN ('student', 'author', 'admin')`) и типы учебных блоков (`type IN ('lecture', 'practice', 'test')`).
3. **Уникальные ключи (`UNIQUE`):** Предотвращают дублирование электронных почт пользователей, повторную запись на один и тот же курс, а также повторную фиксацию прогресса по одному учебному блоку.
